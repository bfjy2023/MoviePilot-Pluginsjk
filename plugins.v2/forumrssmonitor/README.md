# 论坛动态监控

MoviePilot 插件，用于监控论坛 RSS/Atom 动态，默认推送最近 24 小时内的新帖。适合监控 invites.fun 等论坛的标签 RSS。

## 功能特性

- 支持 Atom 和 RSS 2.0
- RSS 地址使用多行文本框配置，一行一个链接
- 默认按发布时间推送最近 24 小时内的新条目
- 检查时间、推送时间和帖子发布时间按东八区显示
- 支持配置 RSS 请求 Cookie，用于访问需要登录态的订阅源
- 保留关键词过滤，标题、作者、摘要任一命中即推送
- 关键词默认留空；留空时只按时间推送
- 首次运行会推送时间范围内的现有条目
- 已见条目会持久化保存，避免重复推送
- 支持定时检查和配置页立即运行一次
- 详情页展示最近检查时间、最近推送记录和最近错误

## 支持版本

- V2 插件目录：`plugins.v2/forumrssmonitor`
- V2 索引：`package.v2.json`
- 当前插件不提供默认版目录

## 安装说明

1. 在 MoviePilot 中进入插件仓库设置，将当前 GitHub 仓库地址添加为插件仓库。
2. 保存并刷新插件市场。
3. 在插件市场安装“论坛动态监控”。
4. 进入插件配置页，填写 RSS 地址列表、关键词和检查间隔。
5. 启用插件后，MoviePilot 会按配置间隔检查 RSS。

## 配置项说明

| 配置项 | 说明 |
| --- | --- |
| 启用插件 | 开启后注册定时检查服务 |
| 发送通知 | 命中关键词后发送插件通知 |
| 立即运行一次 | 保存配置后后台检查一次，并自动关闭该开关 |
| 检查间隔（分钟） | 定时检查 RSS 的间隔，最小 1 分钟 |
| 推送最近小时数 | 按发布时间过滤，默认推送最近 24 小时内的新帖 |
| RSS 地址列表 | 一行一个 RSS/Atom 链接 |
| 请求 Cookie | 可选，RSS 需要登录时填写浏览器 Cookie |
| 关键词 | 逗号或换行分隔；默认留空，留空则只按时间推送 |

## 去重规则

插件会按 RSS 链接保存已见条目 ID。首次运行会推送时间范围内、且命中关键词配置的现有条目，随后同一条目不会重复推送。

## 常见问题

### 没有通知

请确认插件已启用、发送通知已开启、帖子发布时间在“推送最近小时数”范围内。如果填写了关键词，还需要标题/作者/摘要命中关键词。

### RSS 解析失败

请确认 RSS 地址能在浏览器中打开，并返回标准 Atom 或 RSS XML。详情页会展示最近错误。

### invites.fun RSS 访问失败

请在浏览器中登录站点，复制 `flarum_remember=...; flarum_session=...` 这类 Cookie 到“请求 Cookie”配置项。Cookie 失效后需要重新填写。

### 修改关键词后旧帖没有再次推送

已见条目会被记录以避免重复推送。修改关键词后，只会对后续新条目生效。

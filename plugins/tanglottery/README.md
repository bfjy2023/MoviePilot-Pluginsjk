# 不可躺自动抽奖助手

MoviePilot 插件，用于按每日目标次数自动拆解并执行不可躺抽奖。插件提供默认版本和 V2 版本，支持定时执行、立即执行、结果通知、历史记录和抽奖信息展示。

## 功能特性

- 读取 `https://www.tangpt.top/omnibot_lottery.php` 展示单次消耗、今日剩余次数等信息
- 抽奖接口使用 `https://www.tangpt.top/web/omnibot/lottery/draw`
- `count` 可按配置填写，单次接口最多 100 次，大于 100 会自动拆成多批请求
- 支持 MoviePilot 公共定时服务，可在服务列表中查看和手动触发
- 支持配置页“立即运行一次”和详情页/API“立即执行一次”
- 立即执行会在后台运行，页面/API 会先返回“任务已开始”
- 抽奖结束后保存历史记录，并按“发送通知”开关推送结果
- 记录魔力值、折算魔力、上传量、其他奖励和奖品名称汇总
- 支持额度不足、Cookie 失效和连续异常熔断
- 详情页每次打开都会读取不可躺抽奖页面，不使用缓存

## 支持版本

- 默认插件目录：`plugins/tanglottery`
- V2 插件目录：`plugins.v2/tanglottery`
- 默认索引：`package.json`
- V2 索引：`package.v2.json`

两个版本保持同等核心功能，版本号需与索引中的 `version` 保持一致。

## 安装说明

1. 在 MoviePilot 中进入插件仓库设置，将当前 GitHub 仓库地址添加为插件仓库。
2. 保存并刷新插件市场。
3. 在插件市场安装“不可躺自动抽奖助手”。
4. 安装后进入插件配置页，按需填写 Cookie、每日目标总次数和执行周期。
5. 保存配置后启用插件，定时任务会注册到 MoviePilot 服务列表。

## 配置项说明

| 配置项 | 说明 |
| --- | --- |
| 启用插件 | 开启后注册定时服务 |
| 发送通知 | 抽奖任务结束后发送插件通知 |
| 立即运行一次 | 保存配置后启动一次后台抽奖任务，并自动关闭该开关 |
| 每日目标总次数 | 每天计划抽奖总次数，最小值为 1；单次接口最多 100，大于 100 自动拆分 |
| 执行周期 | 5 位 Cron 表达式，例如 `10 2 * * *` |
| 不可躺 Cookie | 可填写包含 `c_secure_pass` 的完整 Cookie |

## Cookie 规则

- 如果插件配置中填写了不可躺 Cookie，插件会优先使用该 Cookie。
- 如果插件配置中留空，插件会读取 MoviePilot 站点管理里域名为 `www.tangpt.top` 或 `tangpt.top` 的站点 Cookie。
- 插件不会修改站点管理中的 Cookie。
- 如果提示 Cookie 失效，请先确认 Cookie 中包含 `c_secure_pass`，再刷新站点 Cookie 或插件 Cookie。

## 使用说明

### 定时任务

启用插件并保存配置后，插件会按 Cron 表达式注册公共定时服务。到达执行时间后会自动抽奖，完成后写入历史记录，并根据通知开关发送结果通知。

### 立即执行

配置页和详情页的立即执行会启动后台任务。点击后接口立即返回“任务已开始”，抽奖继续在后台执行。任务完成后会保存历史记录，并在 `发送通知` 开启时推送结果。

如果已有任务正在运行，立即执行会返回“已有抽奖任务正在执行”，不会重复启动。

### 抽奖信息展示

详情页每次打开都会读取 `https://www.tangpt.top/omnibot_lottery.php`，更新单次消耗、今日剩余次数和最近同步时间。

## 常见问题

### 提示缺少 Cookie

请在插件配置中填写完整不可躺 Cookie，或在 MoviePilot 站点管理中维护 `www.tangpt.top` / `tangpt.top` 的站点 Cookie。

### 提示 Cookie 失效

Cookie 可能过期、缺少 `c_secure_pass`，或不可躺返回登录/权限提示。更新 Cookie 后重新执行即可。

### 提示次数不足

不可躺返回今日剩余抽奖次数不足时，插件会停止本次任务并记录状态，不会继续请求。

### 提示已有任务正在执行

插件同一时间只允许一个抽奖任务运行。等待当前任务结束后再手动执行。

### 页面信息没有变化

详情页会直接请求不可躺抽奖页面。如信息未变化，请查看 `plugins/tanglottery.log` 中的页面请求和解析日志。
